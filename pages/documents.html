border_allGrid System

The grid system in the FreshPlate framework is based on 960px wide grid divided into 12 columns. All rows are wrapped in a 'div'. All columns are wrapped in a 'row' and add up to 12. The grid acts responsively with the following breakpoints:

1. Mobile portrait (Default) - Single column. No need to define. But for those users who want more granular control of mobile screen you can use classes to further split your container.

2. Mobile landscape - @media screen and (min-device-width: 600px)

3. Tablet- @media screen and (min-width: 768px)

4. Large screen- @media screen and (min-width: 1200px)

Tables are very old school. But that is sometimes the very best way to show certain kinds of data. In order to be ADA compliant, you cannot hide your table heading fields. Doing that messes with eRaders for the visually impaired.

On a small device, having more than 2 or three columns becomes a problem. A two column layout, where the column titles repeat on each item is a better option. The sample table has 6 columns. But when you get down to mobile, having those columns crowded in to the screen means it’s impossible to read.

Enter data-label: By moving the th fields off the page by absolute positioning. This allows eReades to see the table headers. We then pull the data-label attribute and place it in front of each Table Data field. This creates a faux 2 column layout.

Go ahead and shrink down the above table and watch what happens. On any device smaller than a tablet, there is a faux 2 column layout, after that it is then back to normal views.

Code Example: <td data-label=Column One>Text in the first table data cell.</td>

lightbulb_outlineHelper Classes

Fresh Plate includes a number of helper classes beyond the usual quick floats and clear fix. To create bold text, for example, add the class ‘bold’, and to create italicized text, add the class ‘italic’.

.onLeft Floats item to the left
.onRight Floats item to the right
.clearfix Clears floated items to avoid spacing issues. Often used in an empty div to stop floats.
.btn_top Creates a floating button that will take the users to the top of the webpage without having to scroll. (Look to your right on this page)
.reverseTransparent
Add to a div to create a special presentation. The ‘reverseTransparent’ class is a shortcut to beautiful headings displayed in front of a masked background. The font family classes can work in conjunction with the ‘reverseTransparent’ class. However, ‘reverseTransparent’ can stand alone. The image behind the text must be applied as a background image to a surrounding div, like a section with its own id.
.hng-p Applying this class to a paragraph creates a hanging paragraph with the first line being bold and italic. This only effects the first line; if a sentence is longer than one line, only the first line words will be different.
.hng-lst Applying this class to an ol or ul formats the list item li to have a first line out-dent, creating a hanging item. Only the first line is affected.
.centerSection Useful for centering block type items in any container. Cannot be used in conjunction with the grid classes.
.notMobile Adding this class sets the display to 'none' for widths less than 768 pixels. This would be useful if you don't want a picture or other item displayed in mobile formats.
.parallax Please see the section "Parallax Scrolling" below.
.scroll Enables smooth scrolling to the desired element on that page.

storageNavigation

The following snippet of code is a basic navbar you can follow to add responsive navigation to any page. For smaller screens, a hamburger menu icon will appear on the top left that you can toggle to open/show the menu links.

HTML:
<nav>
<div class="responsive_nav_icon"></div>
<ul class="responsive_nav">
<li><a href="#">HOME</a></li>
<li><a href="#">ABOUT</a></li>
<li><a href="#">GALLERY</a></li>
<li><a href="#" class="active_nav">CONTACT</a></li>
</ul>
</nav>

To create an active page simply add the class ‘active_nav’ to that page link to show the current active page.

<li ><a href="#" class='active_nav'>HOME</a></li>

Fixed to top

To fix your menu at the top in larger layouts, just add the class top_fixed_nav to your unordered menu list like below.

HTML:
<nav>
<div class="responsive_nav_icon"></div>
<ul class="responsive_nav top_fixed_nav">
<li><a href="#">HOME</a></li>
<li><a href="#">ABOUT</a></li>
<li><a href="#">GALLERY</a></li>
<li><a href="#">CONTACT</a></li>
</ul>
</nav>

Remember to add a margin-top to the main content following the header since it will be overlaid on top of existing content. To account for the height of the header nav, use the CSS below

main { margin-top:4em; }

Fixed to bottom

To fix the header to the bottom use bottom_fixed_nav:

HTML:
<nav>
<div class="responsive_nav_icon"></div>
<ul class="responsive_nav bottom_fixed_nav">
<li><a href="#">HOME</a></li>
<li><a href="#">ABOUT</a></li>
<li><a href="#">GALLERY</a></li>
<li><a href="#">CONTACT</a></li>
</ul>
</nav>

Note: Make sure to give proper margin-bottom to your main content as it will be overlaid by the header at the bottom.

Collapsible Sidebar Navigation

The Freshplate framework also provides classes to have collapsible sidebars. To have your navigation slide in and out from the side use classes “side_nav” and “side_nav_close” as shown in below example. Click here for a working demonstration.

HTML:
<nav class="side_nav " style="left: -500px;">
<div class="side_nav_close"></div>
<ul>
<li><a href="#">HOME</a></li>
<li><a href="#">ABOUT</a></li>
<li><a href="#">GALLERY</a></li>
<li><a href="#">CONTACT</a></li>
</ul>
</nav>
<main class="push_right">
<button class="side_nav_btn">Menu</button>
<div>Main text on page</div>
</main>

The class ‘side_nav_btn’ can be added to any element that you want to click to open the menu. In this example, it's a button.

To close the panel , click on the ‘X’ button on the panel and it will slide back to the left.

By default the sidebar will overlay on top of your main content. You also have the option of pushing your content to the left adding the class ‘push_right’ to the main container that you want to be pushed.

HTML:
<main class="push_right">
<button class="side_nav_btn">Menu</button>
<any content that needs to pushed right>
<...>
</main>

Dropdown

Freshplate also provides an easy mechanism to add submenus to your main navigation using class “submenu”. The submenu class along with the responsive_nav style will provide the default hover dropdown feature on large screens. For responsive small screens, the submenus are included under their corresponding top navlinks like below.

You can overwrite the colors and styles with your own styles. But remember to use dropdowns wisely as they are not the most user-friendly widgets.

HTML:
<nav>
<div class="responsive_nav_icon"></div>
<ul class="responsive_nav" >
<li><a href="#">HOME</a></li>
<li><a href="#">ABOUT</a></li>
<li class="submenu"><a href="#">GALLERY</a>
<ul>
<li><a href="#">Landscape</a></li>
<li><a href="#">Buildings</a></li>
<li><a href="#">Animals</a></li>
</ul>
</li>
<li ><a href="#">CONTACT</a></li>
</ul>
</nav>

Automatic Slideshow

The Carousel is an automatic slideshow which is set to auto-rotate images in a frame with a 2 second pause. The images in carousel need to be optimized to 300x300 px squares. To call the carousel, just create a div with a class of “auto_slideshow”. Add the optmized images in an un-ordered list.

HTML:
<div class="auto_slideshow">
<ul>
<li><img src="../images/cupcake01.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake02.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake03.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake04.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake05.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake06.jpg" alt="cupcake"></li>
</ul>
</div>

The images rotate every 2 seconds and pause for 3 seconds each. The slideshow responds to mouseover events by pausing on hover and continues on unhover.

Gallery with Thumbnail

The FreshPlate framework provides another unique slider format to showcase your images. In this you can show your gallery as a list of thumbnail images in a film-strip-like format and the large images can be seen in the viewer one at a time. You can either use the previous/next icons to scroll through the thumbnail list or directly click on a specific thumbnail to see its larger version. The thumbnail gallery is fully responsive.

Preferred Image Sizes:

Large Size: 300 x 300px

Thumbnails: 50 x 50px

HTML
<div class="gallery_container">
<div class="gallery">
<div class="slider">
<ul>
//Add your own large size images
<li><img src="../images/cupcake01.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake02.jpg" alt="cupcake"></li>
<li><img src="../images/cupcake03.jpg" alt="cupcake"></li>
</ul>
</div> // End slider→
// Slider controls for prev/next icons <div class="sliderCtrls">
<button class="btn_prev" data-dir="prev"></button>
<button class="btn_next" data-dir="next"></button>
</div> // End slider control buttons
</div> // End Gallery
// Thumbnail scroll list set up
<div class="thumb_container">
<div class="thumb_gallery">
<ul> // Add your own thumbnail images
<li>
<a href=""><img src="../images/cupcake01s.jpg" alt="cupcake" height="50px" width="50px"></a>
</li>
<li>
<a href=""><img src="../images/cupcake02s.jpg" alt="cupcake" height="50px" width="50px"></a>
</li>
<li>
<a href=""><img src="../images/cupcake03s.jpg" alt="cupcake" height="50px" width="50px"></a>
</li>
</ul>
</div>
</div>

</div> // End Gallery container

Popup Modal gallery

Freshplate also allows users to showcase a gallery list with a modal pop-up lightbox mechanism without needing to download an external plug-in. You can achieve this by using simple classes.

All you need to do is arrange your images in a container with class ‘popUpContainer’ and add the class ‘popUpImg’ to a div holding the image. Then clicking on the image will produce the blow up of the image in a full size container. Add the description of that image in the ‘alt’ field of the tag and it will get appended as the title caption. In addition, the lightbox is fully responsive so your container should adjust automatically on mobile devices to get the maximum resolution possible.

HTML
<div class="popUpContainer">
<h4>Landcapes</h4>
<div class="popUpImg">
<img alt="Landscape Picture 01" src="images/landscape01.jpg" />
</div>
<div class="popUpImg">
<img alt="Landscape Picture 02" src="images/landscape02.jpg" />
</div>
<div class="popUpImg">
<img alt="Landscape Picture 03" src="images/landscape03.jpg" />
</div>
<div class="popUpImg">
<img alt="Landscape Picture 04" src="images/landscape04.jpg" />
</div>
</div>

move_to_inboxParallax Scrolling

To have a background-image have a parallax effect for a section, simply add the class=”parallax” to that component like below. Then add the background-image for that section in your own style.

HTML <main class="parallax">
<img src=’Forground image here’ alt=’This is where the forgeound image goes>

Or

<div>With Text here.</div>
</main>
CSS .parallax { background-image: url('../images/YourImageHere01.jpeg'); }

*Note: You may use different images on separate containers. The div/section that has the class of "parallax" must only contain the one class, "parallax". By adding a surrounding container unique to the individual space you can have various background images.

Example:

#hero .parallax { background-image: url('../images/YourImageHere01.jpeg'); }

#page2 .parallax { background-image: url('../images/YourImageHere02.jpeg'); }
